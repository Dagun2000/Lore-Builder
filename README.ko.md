# Lore Builder

*가상 세계의 설정을, 문장 하나씩 일관되게 쌓아갑니다.*

Lore Builder는 가상 세계의 역사와 등장인물을 만들어가는 과정에서 설정 일관성을 지켜주는 도구입니다. 사건이나 사실, 관계를 평문으로 서술하고 관련된 엔티티를 `[대괄호]`로 표시하면, 저장되기 전에 지금까지 기록된 모든 설정과 대조합니다:

```
2100년, [쟝]이 [주점]에서 시비가 붙었다.
```

명백한 모순(이미 죽은 인물의 재등장, 파괴된 물건의 재출현, 장소 자체의 설정과 어긋나는 서술)은 자동으로 반려됩니다. 애매한 경우 — 세계관 규칙 위반 가능성, 인물의 기존 설정과의 충돌, 어떤 상관관계 규칙에 비춰봤을 때 앞뒤가 안 맞는 수치 — 는 이해할 수 있는 근거와 함께 화면에 표시되고, 최종 판단은 사람 몫으로 남습니다. 조용히 저장되거나 조용히 무시되는 일은 없습니다.

엔티티와 그 역사는 나란히 관리됩니다: 구조화된 사실(누가 존재하고, 무슨 일이 있었고, 언제였고, 서로 어떻게 연결되는지)은 SQLite에, 자유 서술은 벡터 저장소에 담아서, 정확한 이름을 몰라도 의미로 관련 설정을 찾을 수 있습니다.

## 왜 필요한가

설정이 어느 정도 쌓이고 나면 머릿속만으로 전부 기억하기가 어려워집니다 — 추방된 인물이 슬그머니 수도로 돌아오거나, 소유자가 죽은 뒤에 벼려진 검이 생기거나, 특정 자격이 있어야 가입 가능한 단체에 조건이 안 맞는 인물이 슬쩍 들어가거나. Lore Builder는 이야기를 대신 써주지 않습니다 — 지금까지 확정한 모든 설정을 기억하고 있다가, 새로 들어오는 내용이 그와 어긋나면 나중에 플롯 홀이 되기 전에 미리 짚어줄 뿐입니다.

## 주요 기능

- **자연어 입력** — 사건이나 사실을 평문으로 서술하고 엔티티를 `[대괄호]`로 표시합니다. 인식되지 않는 태그는 신규 엔티티 생성 절차(카테고리 확인, 필수 필드, 문맥에서 추론 가능한 선택 속성까지)로 안내합니다.
- **위키 페이지가 아니라 타임라인** — 모든 사건, 관계, 되돌릴 수 있는 상태(수감, 저주, 추방 등)는 관련된 모든 엔티티에서 참조되는 하나의 타임라인 항목입니다. "현재 상태" 스냅샷은 어디에도 저장하지 않습니다 — 필요하면 그때그때 타임라인에서 계산합니다. 이 세계관 자체에 고정된 "지금"이라는 개념이 없기 때문입니다 — 역사는 어떤 연도든, 어떤 순서로든 추가할 수 있습니다.
- **결정론적 검증** — 사망/파괴 연도 이후 재등장 같은 생애주기 일관성과 종족-수명 대조를 모델 호출 없이 순수 로직으로 처리합니다.
- **LLM 기반 이해** — 문장을 구조화된 사실(누가, 누구에게, 무엇을, 언제)로 변환하고, 신규 엔티티가 등장하면 생애주기 속성, 지속되는 특성, 그리고 별도로 기록할 가치가 있는 사건을 구분해냅니다.
- **새로 들어오는 모든 내용에 대한 모순 검증** — 세계관 규칙 위반 가능성과 엔티티 자신의 기존 설정과의 충돌을 확인합니다: 명시적 제약, 위험도/성격과 행동 톤의 부조화("목숨이 위험한 곳"에서의 여유로운 피크닉), 심지어 상관관계 규칙에 비춰봤을 때 앞뒤가 안 맞는 수치("많을수록 강하다"는 규칙인데 정작 최하위권이면서 스스로는 특출나다고 주장하는 경우 — 단, 규칙 자체에 명시된 예외는 반영합니다)까지 확인합니다. 이 검증은 연도가 있는 사건뿐 아니라 사건 없이 신규 엔티티만 소개하는 문장에도 동일하게 적용되며, 항상 이유와 함께 승인/반려를 선택할 수 있습니다.
- **변경 하나당 결정 하나** — 하나의 변경은 diff 하나(핵심 사실 + 함께 갱신되는 나머지)로 조립되어, 건드린 행마다 한 번씩이 아니라 승인/반려 결정 한 번으로 처리됩니다.
- **직접 관리하는 목록** — 되돌릴 수 있는 상태와 관계 유형(수감, 추방, 동맹 등)은 코드에 박혀있지 않은 평범한 편집 가능 목록입니다. 판단 과정에서 정말로 새로운 유형이 필요해지면, 등록되기 전에 먼저 승인을 요청합니다.
- **웹 GUI** — 같은 자연어 흐름을 쓰는 채팅 모드와, 모든 엔티티를 훑어보는 딕셔너리 모드를 제공합니다. 엔티티를 열면 전체 상세 정보(필드, 관련 기록, 연결된 대상에 대한 집중 관련성 검색)를 보거나, 타임라인 항목을 그 자리에서 수정하거나, 참조 관계까지 자동으로 정리하며 무언가를 삭제할 수 있습니다.
- **원한다면 CLI로도** — 같은 채팅 흐름과, 기존 엔티티를 직접 수정하는 전용 도구를 제공합니다.
- **나중에 검토할 항목 표시** — 검토 중 나온 내용을 저장 여부와 무관하게 다시 살펴볼 항목으로 표시할 수 있습니다. 해당 엔티티 자체가 실제로 고쳐지면 표시는 자동으로 사라집니다.

## 시작하기

### 요구 사항

- Python 3.12 이상
- OpenAI API 키 (추론/교차검증 단계에서만 필요합니다 — 스키마, 저장소, 결정론적 검증은 키 없이도 전부 동작합니다)

### 설치

```bash
python -m venv .venv
source .venv/Scripts/activate    # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 환경 설정

프로젝트 루트에 `.env` 파일을 만듭니다:

```
OPENAI_API_KEY=sk-...
```

### 샘플 세계관 데이터 심기

```bash
python scripts/seed_db.py
```

## 사용법

### 웹 GUI

```bash
streamlit run app.py
```

사이드바 상단에는 어떤 탭에 있든 항상 동작하는 검색창이 있고, 그 아래 두 개의 실제 모드가 있습니다: **채팅** — CLI와 동일한 자연어 파이프라인을 프롬프트 대신 위젯으로 보여줍니다. **딕셔너리** — 모든 카테고리를 훑어보고, 엔티티를 열어 상세 화면을 보거나, 타임라인 항목을 그 자리에서 수정하거나 삭제할 수 있습니다. 세 번째 탭(시각화)은 아직 자리만 마련된 상태입니다.

### 커맨드 라인

```bash
python src/main.py
```

```
Lore Builder CLI. 종료하려면 '종료'를 입력하세요.

입력> 2100년, [미라]가 [은빛도시]에서 산책을 했다.
CREATE timeline: event_2100년_미라가_은빛도시에서_산책했다
  근거: 새 사건 기록: 2100년 미라가 은빛도시에서 산책했다.
  필드: {'year': 2100, 'location': 'loc_silver_city', 'notes': '2100년 미라가 은빛도시에서 산책했다.'}
  함께 갱신되는 엔티티: char_mira, loc_silver_city
저장하시겠습니까? [저장/취소]: 저장
저장 완료: event_2100년_미라가_은빛도시에서_산책했다, char_mira(갱신), loc_silver_city(갱신)
```

새로운 엔티티 태그가 있으면 카테고리 확인과 필수 필드 입력을 먼저 거친 뒤 같은 검토 단계로 이어집니다. 종료하려면 `종료`를 입력하면 됩니다.

이미 존재하는 엔티티를 수정하려면 `python src/detail_panel.py`로 이름이나 id로 찾아서 필드를 선택하고 새 값을 입력합니다 — 저장 전에 이 엔티티가 걸려있는 모든 사건이 나열됩니다.

## 프로젝트 구조

```
schema_registry.yaml       # 엔티티 카테고리, 필드, 타입 정의 — 단일 진실 공급원
status_effects.yaml        # 되돌릴 수 있는 상태와 관계 유형 (수감, 추방, ...)
db/                         # 시드 데이터(YAML 프론트매터가 있는 마크다운) + 생성되는 SQLite/Chroma 저장소
scripts/seed_db.py          # db/*.md를 SQLite + Chroma에 적재
app.py                      # Streamlit GUI 진입점

src/
  config.py                  # 모델 티어 -> 실제 모델명 매핑
  schema.py                   # schema_registry.yaml / status_effects.yaml 로더 + 조회 헬퍼
  storage.py                   # SQLite + Chroma 저장소 계층
  hard_check.py                 # 결정론적 생애주기/수명 검증
  parser.py                      # 정규식 기반 입력 파싱
  mapping.py                      # 엔티티 태그 매칭 + 신규 엔티티 생성
  inference.py                     # LLM 사건/속성 추론
  rag_check.py                      # LLM 기반 세계관 규칙/설정/상태 교차검증
  archivist.py                       # 추론+검증 결과를 검토 가능한 diff로 조립
  pipeline_session.py                 # 파이프라인 상태 머신 (GUI와 CLI가 공유)
  deletion.py                          # 포인터 정리를 포함한 엔티티/이벤트 삭제
  main.py                                # CLI 진입점
  field_update.py                        # 기존 엔티티 필드 수정 로직
  detail_panel.py                         # CLI 엔티티 수정 진입점
  flags.py                                 # "나중에 검토" 기록 관리

tests/                         # pytest 스위트 (다수의 테스트가 OpenAI API를 호출합니다)
```

## 테스트

```bash
pytest
```

LLM을 사용하는 레이어를 검증하는 테스트는 `OPENAI_API_KEY`가 필요하고 실제 API 호출이 발생합니다 — 기본적으로 전체 스위트가 아니라 관련된 파일/테스트만 실행하는 것을 권장합니다. 결정론적 레이어(스키마, 저장소, 하드체크, diff 조립, 플래그)는 키 없이도 전부 통과합니다.
