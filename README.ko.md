# Lore Builder

*판타지 세계관 설정 일관성 검증 파이프라인*

Lore Builder는 세계관에서 방금 벌어진 일을 한 줄로 입력받습니다 — *"2100년, [쟝]이 [주점]에서 시비가 붙었다"* — 그리고 그 사건을 저장소에 반영하기 전에 지금까지 쌓인 모든 설정과 모순되지 않는지 확인합니다. 명백한 모순(이미 죽은 인물의 재등장, 파괴된 아이템의 재출현)은 확인 절차 없이 즉시 반려됩니다. 애매한 경우(세계관 규칙 위반 가능성, 인물의 기존 설정과의 충돌)는 사람이 최종 판단할 수 있도록 화면에 보여줍니다.

두 종류의 지식을 동시에 관리합니다: SQLite에는 구조화된 사실(누가 누구와 어떤 관계인지, 언제 무슨 일이 있었는지, 지금 상태가 어떤지)을, Chroma 벡터 저장소에는 자유 서술 텍스트를 담아서, 정확한 ID를 몰라도 의미로 관련 설정을 찾을 수 있게 합니다.

## 주요 기능

- **자연어 사건 입력** — `[대괄호]`로 감싼 태그를 포함한 평문으로 사건을 입력하면, 파이프라인이 각 태그를 기존 엔티티에 매칭하거나 신규 생성 절차로 안내합니다.
- **결정론적 하드체크** — 생몰년/파괴연도 같은 시간축 일관성 검증과 종족 수명 대조 검증을 LLM 없이 100% 규칙 기반으로 수행합니다.
- **LLM 기반 관계·사건 추론** — 문장을 구조화된 관계로, 필요한 경우 되돌릴 수 있는 상태 변화(수감, 저주, 실종 등)로 변환합니다. 이미 확정된 엔티티만 사용하도록 강하게 제약되어 있어 새 엔티티를 임의로 지어내지 않습니다.
- **RAG 교차검증** — 세계관 규칙 위반 가능성, 인물 설정과의 모순, 현재 상태와의 불일치를 사람이 읽을 수 있는 근거와 함께 표시합니다. 자동으로 반영되는 일은 없습니다.
- **저장 전 전체 검토** — 모든 변경사항은 diff로 조립되어 항목별로 승인받은 뒤에만 저장소에 반영됩니다.
- **기존 엔티티 수정** — 이미 존재하는 엔티티의 필드를 채우거나 고칠 때 쓰는 별도 CLI로, 관련된 모든 기록을 검색해서 저장 전에 함께 보여줍니다.
- **나중에 검토할 항목 표시** — 관련 기록 중 다시 봐야 할 것 같은 항목을 저장 여부와 무관하게 플래그로 남길 수 있습니다.

## 요구 사항

- Python 3.12 이상
- OpenAI API 키 (관계/사건 추론과 RAG 교차검증 단계에서만 필요합니다 — 스키마, 저장소, 하드체크는 키 없이도 전부 동작합니다)

## 설치

```bash
python -m venv .venv
source .venv/Scripts/activate    # Windows: .venv\Scripts\activate
pip install langchain-openai chromadb pyyaml python-dotenv pytest
```

## 환경 설정

프로젝트 루트에 `.env` 파일을 만듭니다:

```
OPENAI_API_KEY=sk-...
```

## 사용법

### 새 사건 기록하기

```bash
python src/main.py
```

```
입력> 2100년, [쟝]이 [주점]에서 술을 마셨다.
[쟝]을(를) char_jang로 인식했습니다.
[주점]을(를) loc_black_goat_inn로 인식했습니다.
[1/2] CREATE timeline: event_...
  승인하시겠습니까? (y/n): y
[2/2] CREATE relationship: rel_...
  승인하시겠습니까? (y/n): y
저장 완료: event_..., rel_...
```

종료하려면 `종료`를 입력하면 됩니다.

### 기존 엔티티 수정하기

```bash
python src/detail_panel.py
```

이름이나 id로 엔티티를 찾고, 필드를 선택하고, 새 값을 입력합니다 — 저장 전에 관련 기록이 검색되어 함께 표시됩니다. 아무 프롬프트에서나 `목록`을 입력하면 지금까지 표시해둔 플래그 목록을 볼 수 있습니다.

### 샘플 세계관 데이터 심기

```bash
python scripts/seed_db.py
```

## 프로젝트 구조

```
schema_registry.yaml       # 엔티티 카테고리, 필드, 타입 정의 — 단일 진실 공급원
status_effects.yaml        # 되돌릴 수 있는 상태의 고정 목록 (수감, 저주, ...)
db/                         # 시드 데이터(YAML 프론트매터가 있는 마크다운) + 생성되는 SQLite/Chroma 저장소
scripts/seed_db.py          # db/*.md를 SQLite + Chroma에 적재

src/
  schema.py                  # schema_registry.yaml 로더 + 조회 헬퍼
  storage.py                  # SQLite + Chroma 저장소 계층
  hard_check.py                # 결정론적 시간축/수명 검증
  parser.py                     # 정규식 기반 입력 파싱
  mapping.py                     # 엔티티 태그 매칭 + 신규 엔티티 생성
  inference.py                    # LLM 관계/사건 추론
  rag_check.py                     # LLM 기반 세계관 규칙/설정/상태 교차검증
  archivist.py                      # 추론+검증 결과를 검토 가능한 diff로 조립
  approval.py                        # 승인 루프 (승인/경고/반려)
  main.py                              # 사건 입력 파이프라인 진입점
  field_update.py                      # 기존 엔티티 필드 수정 로직
  detail_panel.py                       # 엔티티 수정 진입점
  flags.py                               # "나중에 검토" 기록 관리

tests/                         # pytest 스위트 (일부 테스트는 OpenAI API를 호출합니다)
```

## 테스트

```bash
pytest
```

LLM을 사용하는 레이어를 검증하는 테스트는 `OPENAI_API_KEY`가 필요합니다. 결정론적 레이어(스키마, 저장소, 하드체크, diff 조립, 플래그)는 키 없이도 전부 통과합니다.
